# 新規機能開発ワークフロー

以下の段階的な手順に沿って新規機能の開発を進めます。

## 1. 事前準備と仕様の確認

- `Gemini.md` が存在する場合は、最初に必ず読み込んでプロジェクトのルールや文脈を把握してください。
- ユーザーから渡された仕様書や要件を確認し、少しでも疑問に感じたこと、エッジケース、不明確な点があれば**必ずユーザーに質問**してください。
- ⚠️ **仕様書がない場合**: ユーザーに「どのような機能を開発したいか（要望）」を詳しくヒアリングし、その要望をもとにAI側で仕様書（または要件のまとめ）を作成してください。作成後、ユーザーに確認を取ってください。

## 2. デザインシステムと計画書の作成

- 開発する機能に UI/UX が含まれる場合は、まず `ui-ux-pro-max` スキルを活用してデザインシステムを定義し、プロジェクトのトンマナを統一します。
  - プロダクトの種類、業界、スタイルのキーワード（例: "SaaS dashboard minimalist"）を特定します。
  - 以下のコマンドを実行してデザインシステムを生成し、保存します（`--persist` フラグを使用）。
    ```bash
    python3 .agents/skills/ui-ux-pro-max/scripts/search.py "キーワード" --design-system --persist -p "プロジェクト名"
    ```
  - 生成された `design-system/MASTER.md` を確認し、必要に応じてページ固有のオーバーライドを作成します。
- 仕様に基づき、実装に向けた計画書（具体的なステップ、作成・変更するファイル、テストの方針、定義したデザインシステムの適用方法など）を作成してください。
- 作成した計画書をユーザーに提示し、問題がないか**確認（合意）をとってください**。

## 3. テストコードの実装

- 計画書と仕様書にある要件をテストコードに落とし込みます。
- テストコードを作成し、要件を網羅できているか確認してください。

## 4. プロダクトコードの実装とUI品質の確認

- テストコードに問題がないと判断できたら、実際のプロダクトコードの実装に移ってください。
- 実装時には、定義したデザインシステム（`design-system/MASTER.md` 等）のルールを厳格に守ってください。
- 実装完了後、以下の点を含め `ui-ux-pro-max` のチェックリストに従って品質を確認します。
  - アイコンに絵文字を使用していないか（SVGを使用）
  - すべてのクリック可能要素に `cursor-pointer` が設定されているか
  - ホバー状態や遷移（150-300ms）が適切に設定されているか
  - レスポンシブ対応やアクセシビリティ（alt属性等）が考慮されているか
- テストを実行してパスすることを確認し、必要に応じてリファクタリングを行います。
