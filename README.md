LogFoは、日々のプログラミング、ソフトウェアに関する学習の軌跡（ログ）を自動で集約・可視化し、それを信頼できるポートフォリオに変換することで、学習者のモチベーション維持とキャリア目標の達成を支援するSassプロダクトです。

---


### 1. 学習の自動可視化（ダッシュボード機能）

学習者の活動をAPI連携によって自動的に集約し、学習の「量」と「成果物」を一元管理します。

* **コーディング活動:** GitHub（コミット数）、WakaTime（実稼働時間）
* **アルゴリズム学習:** AtCoder / LeetCode（問題数、レーティング）
* **技術アウトプット:** Zenn / Qiita（記事数、LGTM数）

---

### 2. 内発的動機付け（学習の「楽しさ・達成感」の仕組み）

日々の学習プロセスをゲーミフィケーション化し、内面的な達成感を刺激します。

* **スキルツリー / コンピテンシーマップ:**
    学習活動（Zenn執筆、書籍読了、関連リポジトリの作成）に応じて、スキルノード（例: 「Go言語の非同期処理」）が「解放」されていく。
* **ポートフォリオ・ギャラリー:**
    作成したリポジトリや執筆したZenn記事がカード形式で自動的に蓄積され、具体的な「成果物」を実感できる。
* **学習ロードマップ:**
    選択した目標ロール（例: 「Webエンジニア」）に対する学習の進捗バーが、日々の活動によって自動で埋まっていく。

---

### 3. 外発的動機付け（「目標達成」の仕組み）

学習者のキャリア目標と日々の学習を直結させ、学習の「意味」を強化します。

* **目標設定機能:** 「入社したい企業」「目標年収」を設定。
* **目標企業の動向トラッキング:**
    設定した目標企業のエンジニアが発信するZenn, Qiitaの最新記事フィードをダッシュボードに表示。
* **スキルギャップの可視化:**
    「目標企業の求めるスキルセット」と「（機能1で自動取得した）現在の学習状況」を比較し、「あなたには『AWS』と『TypeScript』が不足しています」といった形でギャップを可視化する。

---

### 4. （新機能）ポートフォリオ自動生成・公開機能

学習者が積み重ねた全てのログを、ボタン一つで「客観的なポートフォリオ」として外部に公開できる機能です。

* **プライバシー保護（オプトイン）:**
    * 全ての学習ログは**デフォルトで「非公開」**となります。
    * ユーザーが「ポートフォリオとして公開する」と明示的に許可（オプトイン）した場合にのみ、公開用ページが生成されます。
* **公開範囲のカスタマイズ:**
    * ユーザーは「WakaTimeの総時間グラフは公開するが、詳細な時間帯は隠す」「このプライベートリポジトリは公開しない」など、**項目ごとに公開・非公開を詳細に設定**できます。
* **採用担当者向けサマリー:**
    * 生成される公開ページは、単なるログの羅列ではなく、採用担当者が見やすいように**サマリー（要約）**されます。
    * 例：「直近3ヶ月の学習時間は合計XXX時間」「最も得意な言語はTypeScript（XXX時間）」「AWSに関するスキルツリーを解放済み」「ZennでXXX LGTMを獲得した記事あり」
* **信頼性の担保:**
    * 自己申告のスキルシートとは異なり、API連携による「生データ」に基づいているため、**学習意欲や継続性の客観的な証明**として機能します。

---

### 5. 想定されるユーザー体験（キャリアへの接続）

これまでの学習ループに加え、キャリアに接続する流れが完成します。

1.  **学習ループ:** （これまでの議論通り）「目標設定」→「ギャップ認識」→「学習実行」→「達成感（スキルツリー解放）」のサイクルを回す。
2.  **就活フェーズ:** 転職・就職活動を開始するタイミングで、ユーザーが「ポートフォリオ公開」ボタンを押す。
3.  **公開設定:** 公開したい項目（GitHub, Zenn, WakaTimeのサマリ、解放済みスキルツリーなど）を選択し、公開用URLを生成する。
4.  **応募:** 履歴書や職務経歴書に、この**「学習ログ・ポートフォリオURL」**を記載する。
5.  **採用担当者の確認:** 採用担当者はURLにアクセスし、候補者が「どのような目標を持ち」「どれだけの時間と熱意をかけて」「体系的に学習してきたか」を客観的なデータで確認する。
