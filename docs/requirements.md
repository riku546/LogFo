# LogFo 全体要件定義書 (Overall Requirements Definition)

## 1. プロジェクト概要

LogFoは、エンジニアなどの目標達成（理想の企業への入社、スキルの習得など）を目指すユーザーをサポートするためのプラットフォームです。
ユーザー自身の現在地と目標を基にLLMによるロードマップ生成を行い、その計画に沿った日々の活動記録、外部サービス（GitHub等）と連携した学習の可視化、そして最終的な活動のまとめとポートフォリオ生成までを一貫してサポートすることで、目標達成までのフローを伴走します。

## 2. 解決する課題と目標達成フローの適合性

エンジニアを目指すユーザーが直面する「学習計画の策定」「モチベーションの維持」「成果の効果的なアピール」という課題（詳細は[problem_analysis.md](./problem_analysis.md)参照）を、一連のフローを通して包括的かつ強力に解決します。

## 3. システムの全体フローと機能要件

LogFoは以下の4つの主要なステップ（フロー）で構成され、ユーザーの目標到達からアピールまでをシームレスに繋ぎます。
各フローの具体的な機能要件や仕様については、それぞれのリンク先ディレクトリの設計ドキュメントを参照してください。

### Step 1: LLMを活用したロードマップ生成

自身の「現在地（スキル・経験）」と「目的地（志望企業・目標）」を入力し、ギャップを埋めるための具体的な学習マイルストーンやタスクを含んだロードマップをLLMを利用して自動生成します。何をすべきかわからないという初期のハードルを下げます。
👉 **詳細な要件:** [ロードマップ生成機能 (Roadmap Generation)](./specs/roadmap_generation/requirements.md)

### Step 2: 学習の継続と可視化 (実行フェーズ)

生成された計画の実行をサポートし、挫折を防ぐための機能です。以下の2つの側面からモチベーションの維持を図ります。

- **Step 2-1: 日々の活動・進捗記録**
  ロードマップ上の各ステップ（タスク）に対して、日々の学習内容や気づきを記録（テキスト等）し、進捗を管理します。これらの記録は後のふりかえり・自己PRの基盤データとなります。
  👉 **詳細な要件:** [日常的な活動・進捗記録機能 (Activity Tracking)](./specs/activity_tracking/requirements.md)

- **Step 2-2: 外部データ連携とダッシュボード化**
  GitHub、WakaTime、Zenn、Qiitaなどの外部サービスにおける活動を自動で収集・統合し、ヒートマップやグラフとして可視化します。客観的なデータで継続を可視化し、自己効力感を高めます。
  👉 **詳細な要件:** [外部データ連携・ダッシュボード機能 (Dashboard)](./specs/dashboard/requirements.md)

### Step 3: LLMによる活動記録の総括・要約 (特定の目標単位)

蓄積された「Step 2-1 の日々の活動記録」と「Step 2-2 の外部サービスの客観データ」をLLMに読み込ませ、**特定の目標（ロードマップ）に対して「どのような壁にぶつかり、どう解決し、どう成長したか」というストーリー性のある自己PRや活動サマリー**を自動生成します。
※ ここで生成される「サマリー」は、1つの特定の目標やプロジェクトに対する振り返りの文章（テキストなどのコンテンツ）であり、最終的なWebページ（ポートフォリオ）に組み込むための一要素（部品）となります。
👉 **詳細な要件:** [活動サマリー生成機能 (Activity Summary)](./specs/activity_summary/requirements.md)

### Step 4: 最終ポートフォリオの生成と共有

生成されたサマリー、プロフィール情報、外部サービスのグラフなどの実績データを組み合わせて、採用担当者等に見せるための「ポートフォリオページ（Webページ）」を構築し、パブリックリンクとして共有できるようにします。
👉 **詳細な要件:** [ポートフォリオ生成・共有機能 (Portfolio Generation)](./specs/portfolio_generation/requirements.md)
