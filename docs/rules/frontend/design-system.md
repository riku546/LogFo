# LogFo Design System Guidelines

このドキュメントは、LogFoプロジェクトにおけるフロントエンドのデザイン定義と実装ルールを定めたものです。
ユーザーヒアリングに基づき、**信頼感のあるプロフェッショナルなイメージ**をベースに、**Glassmorphism**のエッセンスを取り入れたデザインを採用します。

## 1. デザインコンセプト

- **パーソナリティ**: 信頼、プロフェッショナル、清潔感 (A)
- **カラー戦略**: ホワイトを基調とし、クールな青をアクセントにする
- **スタイル**: Glassmorphism (透明感、ぼかし、レイヤー)
- **テーマ**: ライトモードをメインとし、ダークモードへの切替をサポート

## 2. カラーパレット

| 役割                  | ライトモード (Hex/RGBA)    | ダークモード (Hex/RGBA)     | 説明                                     |
| --------------------- | -------------------------- | --------------------------- | ---------------------------------------- |
| **Background**        | `#FFFFFF`                  | `#0F171C`                   | 基本背景色。ダークは視認性のため微調整。 |
| **Accent (Primary)**  | `#2563EB`                  | `#2563EB`                   | クールな青。ボタン、強調部分に使用。     |
| **Foreground (Text)** | `#0F172A`                  | `#F8FAFC`                   | 高いコントラストを保つテキスト色。       |
| **Glass Background**  | `rgba(255, 255, 255, 0.8)` | `rgba(30, 41, 59, 0.7)`     | カードの背景。背後をぼかして質感を出す。 |
| **Glass Border**      | `rgba(255, 255, 255, 0.5)` | `rgba(255, 255, 255, 0.15)` | エッジを強調し輪郭をはっきりさせる。     |

## 3. タイポグラフィ

- **Heading (見出し)**: `Poppins`
  - 変数: `--font-poppins`
  - モダンで幾何学的な形状がプロフェッショナルかつクリーンな印象。
- **Body (本文)**: `Inter` または `Geist Sans`
  - 変数: `--font-sans` (フォールバック含む)
  - 読みやすさを重視。

## 4. Glassmorphismの実装ルール

視認性と立体感を確保するため、以下のユーティリティ（`.glass`）を適用します。

### ライトモード

- **Blur**: `backdrop-blur(16px)`
- **Shadow**: `box-shadow: 0 8px 32px 0 rgba(31, 38, 135, 0.07)`
- **Border**: 明い白の境界線で清潔感を出す。

### ダークモード

- **Blur**: `backdrop-blur(16px)`
- **Shadow**: `box-shadow: 0 8px 32px 0 rgba(0, 0, 0, 0.4)`
- **Border**: 控えめな白の境界線で輪郭を出す。

## 5. UIコンポーネントの基本原則

- **Radius (角丸)**: `rounded-xl` (12px) または `rounded-2xl` (16px) を標準。
- **Interaction**:
  - クリック可能な要素には必ず `cursor-pointer` を付与。
  - ホバー時は `transition-all duration-300` で滑らかに変化。
- **Icons**:
  - `Lucide React` や `Heroicons` のSVGアイコンで統一します。

## 6. テーマ管理

- `ThemeProvider` (React Context) により、アプリケーション全体で `light` / `dark` を切り替えます。
- HTML要素の `.dark` クラスによって CSS 変数が上書きされます。
