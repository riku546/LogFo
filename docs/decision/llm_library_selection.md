# Architecture Decision Record: LLMライブラリ/サービスとしてのハイブリッド構成採用

**タイトル:** LLMライブラリ/サービスとしてのハイブリッド構成採用
**ステータス:** Accepted

### Context (背景と要件)

LLMライブラリ/サービスの選定にあたり、料金コスト、無料プラン、モデルの豊富さ、Cloudflareとの相性、将来性・保守を比較観点とした。

### Options Considered (検討した選択肢)

比較対象は以下の3つである。

- OpenRouter
- Cloudflare Workers AI
- 単一プロバイダーSDK (gen ai sdk等)

### Decision (決定事項)

## 結論：ハイブリッド構成の採用

### 採用方針

**コスト最適化を重視し、OpenRouter + Cloudflare Workers AI のハイブリッド構成を採用する。**

### Consequences (結果・影響)

### この構成のメリット

1. **コスト効率**
   - 開発時はほぼ無料で回せる
   - 本番でも大部分をWorkers AIで処理し、高額なフロンティアモデルは必要時のみ

2. **Cloudflareスタックとの親和性**
   - Workers AIはネイティブbindingsで最高のDX
   - Hono + Cloudflare Workers + Next.js構成と完全に統合

3. **柔軟性**
   - OpenRouter経由なら400+モデルにアクセス可能
   - 処理に応じて、軽量モデルと高度なモデルを切り替えられる

4. **スケーラビリティ**
   - Workers AIのサーバーレス特性で自動スケール
   - OpenRouterのフォールバック機能で可用性確保

## 比較表

| 観点 | OpenRouter | Cloudflare Workers AI | 単一プロバイダーSDK (gen ai sdk等) |
|------|-----------|----------------------|-------------------------------------|
| **料金コスト** | プロバイダー元価格 + クレジット購入時5.5%手数料 | $0.011/1,000 Neurons (超過分のみ) | SDK自体は無料、プロバイダーの従量課金のみ |
| **無料プラン/トークン量** | 無料モデル50リクエスト/日 ($10購入で1,000/日に)、DeepSeek等一部無料 | **10,000 Neurons/日が無料** (毎日リセット) | プロバイダー依存 (OpenAI: 無料枠なし、Anthropic: 約$10/月の無料枠) |
| **モデルの豊富さ** | **400+モデル** (GPT, Claude, Gemini, Llama等すべて) | オープンソースモデルのみ (Llama, Mistral, Gemma, DeepSeek等) ※GPT-4/Claude不可 | 単一プロバイダーのモデルのみ |
| **Cloudflareとの相性** | ◎ (CF経由でも利用可、エッジ最適化済み) | **◎◎** (完全ネイティブ統合、bindings直接利用) | △～○ (SDK依存、一部制約あり) |
| **将来性・保守** | ◎ (a16zパートナー、25万+アプリ導入) | ◎ (Cloudflare本体が開発、継続的更新) | プロバイダー依存 (gen ai sdkは活発に開発中) |

## 開発環境

```
┌─────────────────────────────────────────────────┐
│                  開発時の構成                    │

│  OpenRouter (無料モデル)                         │

│  - Gemma 3                                      │
│  → 機能開発・プロンプト調整に使用                  │

│  - 10,000 Neurons/日 (無料)                      │

               コスト: ほぼ無料
```
