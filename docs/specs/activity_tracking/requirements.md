# 日常的な活動・進捗記録機能 要件定義書 (Activity Tracking Requirements)

## 1. 機能概要

生成（また作成）されたロードマップの各ステップ（タスク）に対して、ユーザーが日々の学習内容、学んだこと、気づきなどを記録する機能。この記録は学習履歴として蓄積され、後の自己アピール（ポートフォリオ）やふりかえり（Summary Generation）に活用されます。

## 2. 対象ユーザー

- ロードマップに沿って学習を進めるユーザー
- 学習の軌跡を残し、定期的に振り返りを行いたいユーザー

## 3. ユースケース (User flow)

1.  **タスクの選択:** ユーザーが自身のロードマップから現在取り組んでいるタスクを選択する。
2.  **ステータスの更新:** タスクを「未着手」「進行中」「完了」にステータス変更する。
3.  **記録の作成:** タスクに関連する「日報」や「メモ」のような形で、学習内容をMarkdown形式で記録する。
4.  **記録の確認・保存:** 作成した記録を保存し、タイムラインやタスク詳細画面で過去の記録を閲覧する。

## 4. 機能要件 (Functional Requirements)

### 4.1. ロードマップ連動機能 (Frontend)

- ユーザーの進捗が見えるように、各ステップ（マイルストーン、タスク）を一覧化（ToDoリスト、カンバン等）する。
- タスクをクリックすると詳細が開き、そこに日々の活動を記録できる入力エリア（リッチテキスト または Markdownエディタ）を表示する。
- タスクのステータス変更（TODO -> IN_PROGRESS -> DONE）。

### 4.2. 活動記録の作成・編集 (Frontend)

- **Markdownエディタ:** コードスニペットのハイライトやリンク、チェックリストをサポートするリッチエディタ。
- **入力項目:**
  - 学習した内容（Must）
  - 所感や気付き（任意）
  - 参考にした資料のURL等（任意）
  - 関連する外部プラットフォームのリンク（例えばGitHubのリポジトリやZennの記事リンクなど）（任意）
- 過去の記録（同じタスクに対する過去の日報）をタイムライン形式で表示。
- **LLMによる記録作成支援:** 「何を書けばいいかわからない」ユーザー向けに、LLMを活用した「今日の学びのアウトライン生成」や「質問に答える形式での記録作成支援」機能（ウィザード形式）を提供する。

### 4.3. 記録データの保存・管理 (Backend)

- タスクに紐づく形で活動記録（Activity Log/Note）をデータベースに保存する。
- 記録データのCRUD操作を提供するAPI。
- 記録にはタイムスタンプ（作成日時、最終更新日時）を付与する。

## 5. 非機能要件 (Non-Functional Requirements)

- **UX:** 日々の記録は「面倒くさい」と感じられやすい作業のため、エディタは軽量で、かつ「記録した！」という達成感を少しでも感じられるようなUIフィードバック（マイクロアニメーションなど）を提供する。
- **データ形式:** 記録内容はプレーンテキストとしてデータベースに保存し（XSS対策はフロントエンドで行う）、将来的にLLMが読み込みやすい形式（構造化テキスト等）を維持する。
