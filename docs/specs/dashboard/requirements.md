# 外部データ連携・ダッシュボード機能 要件定義書 (External Data & Dashboard Requirements)

## 1. 機能概要

GitHub、WakaTime、Zenn、Qiita、AtCoderといった外部サービスから学習・開発活動のデータを自動（または手動同期）で取得し、それらを総合的な「ヒートマップ（草）」や各種グラフとして可視化するダッシュボード機能。日々の活動が可視化されることで、ユーザーの学習継続へのモチベーションを維持します。

## 2. 対象ユーザー

- 自分の学習量やアウトプットを可視化したいユーザー
- 他のサービスに分散している活動記録を一箇所にまとめたいユーザー
- 目標達成のために「どれくらい頑張ったか」を定量的に確認したいユーザー

## 3. ユースケース (User flow)

1.  **連携設定:** ユーザーが各プロバイダ（GitHub、WakaTime等）の認証情報やアカウントIDを設定する。
2.  **データ取得:** バックグラウンド処理により、各サービスから活動データを取得する。
3.  **データ可視化 (ダッシュボード):** ユーザーがダッシュボードを開き、すべてのプラットフォームの活動が統合されたヒートマップやサマリーグラフを確認する。
4.  **詳細確認:** 特定の日付をクリックし、その日の具体的な活動内容（コミット内容、コーディングした言語、執筆した記事など）を確認する。

## 4. 機能要件 (Functional Requirements)

### 4.1. 外部サービス連携 (Frontend/Backend)

- **OAuth認証・アクセストークン管理:** GitHub, WakaTimeのようなOAuth対応サービスのアクセストークンを安全に取得・保存する。
- **アカウントID連携:** Zenn, Qiita, AtCoderなどのRSSや公開APIを利用するサービスについては、ユーザーIDを入力して連携設定を行う機能。
- **連携状態の管理:** 現在連携中のサービス一覧を表示し、連携の解除や再認証を行える設定画面。

### 4.2. データ取得・集約バッチ処理 (Backend)

- 各プロバイダのAPIを定期的にポーリング（例: 1日1回、あるいはログイン時のオンデマンド）し、活動データ（コミット数、記事投稿、コーディング時間等）を取得する。
- 取得したデータをLogFoの統一されたActivityデータモデル（いつ、どこで、何を、どれくらいしたか）に変換し、データベースに保存する。
- **Rate Limit対応:** 外部APIの呼び出し制限（Rate Limit）に配慮した設計（キューイング、キャッシュなど）。

### 4.3. ダッシュボード・データ可視化 (Frontend)

- **プロバイダー別活動ウィジェット:** ユーザーの「焦り」や「継続モチベーション」を刺激するため、プロバイダーごとに以下の要素を組み合わせたウィジェットを表示する。
  - **活動バッテリー（エナジー残量）:** 最新の活動日を基準に、活動しない日が続くと残量が減少（例: 1日未活動で-20%）するメーター。0%にならないように活動を促す。
  - **直近活動グラフ:** 直近10日間の日別活動量をシンプルな棒グラフで表示し、短期的なトレンドを可視化する。
- **活動サマリー・統計情報 (グラフ表示):**
  - 総活動時間、記事執筆数、コミット数などの数値サマリー。
- **日別詳細ビュー:** ウィジェットの特定の日付（棒グラフ等）を選択した際に、その日の具体的な活動内容（コミット内容、記事投稿など）をリスト表示する。

## 5. 非機能要件 (Non-Functional Requirements)

- **セキュリティ:** 保存するOAuthのアクセストークンやシークレット情報は暗号化してDBに保存し、バックエンド以外からはアクセスできないようにする。
- **パフォーマンス:** 全アクティビティログをフロントエンドで描画する際、集計クエリが重くならないように、日ごとの集約データ（マテリアライズドビュー等）を作成しておくこと。
- **UI/UX:** データが存在しない（連携したて）の場合でも、「空の状態（Empty State）」を美しく見せ、ユーザーに連携を促すデザインを適用する。

## 6. 特筆事項 / 検討課題

- 自身の活動データだけではなく、他のユーザーの活動データと比較できる機能も提供する。（Rate limitに注意）
- 将来的には、目標としている企業で働くエンジニアの平均アクティビティ量等と比較する「ベンチマーク表示」もこのダッシュボード上で対応する。
