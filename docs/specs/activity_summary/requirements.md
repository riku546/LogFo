# 活動サマリー生成機能 要件定義書 (Activity Summary Generation Requirements)

## 1. 機能概要

蓄積された「日々の活動記録 (Activity Tracking)」と「外部サービスの客観データ (Dashboard)」をLLM（Large Language Model）に読み込ませ、これまでの学習の軌跡や目標に対する達成度、ストーリー性のある自己PRを自動（または半自動）で要約・生成する機能。

## 2. 対象ユーザー

- 「これまで何をやってきたか」を文章としてまとめたいが、うまく言語化できないユーザー
- 定期的な振り返り（月報、四半期レビューなど）を行いたいユーザー
- 就職・転職活動に向けた職務経歴書・自己PRのベース（ドラフト版）を手間なく作成したいユーザー

## 3. ユースケース (User flow)

1.  **対象マイルストーンの選択:** ユーザーがロードマップ内から、サマリーを生成したい「マイルストーン（中間目標）」を選択する。
2.  **サマリー生成の指示:** サマリー生成ボタンをクリックし、LLMに指示（プロンプト）を出す。
3.  **LLMによる生成・処理:** システムがそのマイルストーンに紐づく活動記録（タスクのログ）と、マイルストーンの開始日〜終了日時点の外部データをプロンプト化し、LLMに送信。要約情報（Markdown）を受け取る。
4.  **編集・保存:** ユーザーが生成されたサマリー内容を確認および編集（手直し）し、自分の「ストーリー」として保存する。

## 4. 機能要件 (Functional Requirements)

### 4.1. データ抽出とプロンプト構築 (Backend)

- ユーザーが選択した「マイルストーン」に紐づくすべてのタスクの「活動記録（ノート、学び）」と、そのマイルストーンの期間（最初のログ日時から最後のログ日時）に該当する「外部サービスのアクティビティデータ（例: WakaTimeのxx時間のPythonコーディング、GitHubでのxxコミット）」をデータベースから抽出する。
- 抽出したデータをLLMに適した形式（箇条書き、JSONのリストなど）に整形し、「これらを元に、このユーザーがどのような壁にぶつかり、どう解決し、どう成長したかの自己PR・学習サマリーとして要約してください」という指示（システムプロンプト）と組み合わせてAPI（OpenAI or Anthropic）を呼び出す。

### 4.2. サマリー表示および編集機能 (Frontend)

- 生成中のスピナー・ローディング表示。
- 生成されたサマリーコンテンツ（Markdown形式）を見出し等で整形して表示するプレビュー機能。
- 生成されたサマリーをユーザーが手動で編集・加筆・修正できるエディタ（リッチテキスト または Markdown）。
  - 生成されたサマリーは後からアクセスできるようにシステム（DB）に保存する。

### 4.3. サマリー生成フォーマットの選択 (Frontend/Backend)

- ユーザーの目的に応じて出力のフォーマット（トーン＆マナー）を変更できる機能（例: 「就活用の自己PR風」「エンジニア向けの月報風」「カジュアルな振り返り」等）。

## 5. 非機能要件 (Non-Functional Requirements)

- **プロンプトサイズの最適化:** 短期間でも大量の活動記録がある場合、LLMのコンテキストウィンドウ（トークン制限）を超過する可能性があるため、事前に簡単な要約処理を挟むか（Map-Reduceアプローチ）、必要な情報だけを抽出（EmbeddingsとRAGの利用等）するアーキテクチャを検討する。

## 6. 特筆事項 / 検討課題

- ここで生成・編集されたサマリー（ストーリー）テキストが、ポートフォリオ生成の際のメインコンテンツ（自己紹介・強み）として活用されるため、ポートフォリオシステム（Step 4）とのシームレスなデータ連携設計を行う。
