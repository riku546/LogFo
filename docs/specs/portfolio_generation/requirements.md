# ポートフォリオAI文章生成・共有機能 要件定義書 (Portfolio AI Narrative Requirements)

## 1. 機能概要

本機能は、ユーザーのプロフィール情報と選択したサマリー、自己PR下書きをもとに、LLMがポートフォリオ向け文章（自己PR・強み・学び・将来）を生成し、公開ページで共有できるようにする機能です。

現行の「公開ページでロードマップ/サマリー本文を直接表示する構成」は廃止し、AI生成文章を中心とした構成へ移行します。

## 2. 対象ユーザー

- 就職・転職活動中のエンジニア
- 学習成果を対外的に分かりやすく提示したいユーザー
- 手動で自己PR文章を組み立てる時間を削減したいユーザー

## 3. ユースケース (User flow)

1. ユーザーがビルダー画面（`/portfolio`）を開く。
2. プロフィール情報（表示名、経歴、スキル等）を入力/更新する。
3. AI文章生成セクションで、生成入力用サマリー（最大5件）を選択し、必要に応じて自己PR下書きを入力する。
4. ユーザーが「4項目一括生成」を実行し、生成結果を確認する。
5. 必要に応じて項目ごと（自己PR・強み・学び・将来）に再生成または手編集する。
6. 既存の保存ボタンでポートフォリオを保存する。
7. 公開ページ（`/portfolio/{slug}`）で `PR・強み` タブに生成文章を表示し、外部共有する。

## 4. 機能要件 (Functional Requirements)

### 4.1. ポートフォリオ設定・構築機能

- プロフィール設定:
  - 表示名（必須）
  - 自己紹介
  - アバターURL
  - SNSリンク（GitHub, X, Zenn, Qiita, AtCoder, Website）
  - 経歴ストーリー（複数件の追加/編集/削除）
  - スキル（文字列タグの追加/削除）
- 公開設定:
  - Slug必須、英小文字/数字/ハイフンのみ、最大50文字
  - 公開/非公開切替

### 4.2. AI文章生成機能

- 生成対象は以下4セクションとする:
  - 自己PR (`selfPr`)
  - 強み (`strengths`)
  - 学び (`learnings`)
  - 将来 (`futureVision`)
- 4セクションはすべて任意表示（空文字の場合は公開ページで非表示）。
- サマリー選択UI:
  - ユーザー自身のサマリーから選択可能
  - 選択上限は5件
- 補助入力:
  - 自己PR下書き（自由記述）を入力可能
- 生成実行条件:
  - 「サマリー選択が1件以上」または「自己PR下書きが非空」のいずれかを満たすこと
- 生成操作:
  - 4項目一括生成
  - 項目別再生成
- 生成文体:
  - 就活/転職向けフォーマル
  - 各項目 200-350字を目安

### 4.3. 公開表示機能

- 公開URLは `logfo.app/portfolio/{slug}` 形式。
- 公開ページの2ndタブ名は `PR・強み`。
- `PR・強み` タブにはAI生成4セクションのみ表示する。
- 旧ロードマップ/サマリー本文表示は行わない。
- サマリーは生成入力用データとしてのみ利用し、公開本文として表示しない。

### 4.4. データ保存

- 保存対象は `PortfolioSettings` に保持する。
  - `profile`
  - `generation`（`selectedSummaryIds`, `selfPrDraft`）
  - `generatedContent`（4セクション）
- 生成結果の確定タイミングは既存の保存ボタン押下時とする。

## 5. 受け入れ基準 (Acceptance Criteria)

- 必須入力条件:
  - ポートフォリオ保存時は表示名・Slugが必須
  - 生成API実行時は、サマリー未選択かつ自己PR下書き空の組み合わせを拒否
- 表示条件:
  - 公開ページでは4セクションの非空項目のみ表示
  - 旧ロードマップ/サマリー本文は表示されない
- エラー条件:
  - サマリー選択が6件以上の場合はバリデーションエラー
  - ユーザーが所有しないサマリーIDを指定した場合はエラー

## 6. 非機能要件 (Non-Functional Requirements)

- レスポンシブ対応: モバイル/デスクトップで崩れないこと。
- UX:
  - 生成中状態（ローディング）を明示すること。
  - 項目別再生成時に対象項目が分かること。
- パフォーマンス:
  - サマリー選択UI用データ取得はN+1を避けた一覧取得APIを使うこと。

## 7. MVP対象外（将来拡張）

- テーマ切替（テンプレート複数化）
- 外部活動データウィジェット統合
- CTA（面談申込、問い合わせ）導線
- OGP画像動的生成
