# ポートフォリオ生成・共有機能 要件定義書 (Portfolio Generation Requirements)

## 1. 機能概要

これまで蓄積してきた「活動記録（Activity Data）」、生成した「活動サマリー（Summary）」、および「外部連携データ（GitHubの草、WakaTimeの言語グラフなど）」を組み合わせて、採用担当者が見やすい「Webページ（ポートフォリオ）」を半自動（LLM等でアシストされながらの設定）または手動で生成・公開する機能。

## 2. 対象ユーザー

- 就職・転職活動中のエンジニア（カジュアル面談等で企業に提出したいユーザー）
- これまでの軌跡を社内外（副業など）でアピールしたいユーザー
- Notionなどでポートフォリオを手作業で作ることに手間に感じているユーザー

## 3. ユースケース (User flow)

1.  **ポートフォリオ新規作成:** ユーザーが「ポートフォリオ作成」ボタンをクリックする。
2.  **要素の選択（カスタマイズ）:** 表示するプロフィール情報、強調したいスキル実績（連携データ）、LLMが作成したサマリーなどを選択・調整する。
3.  **プレビュー・自動レイアウト:** 選択内容に基づき、システム（またはLLMによる提案）が自動的に見やすく配置（レイアウト）したプレビュー画面を表示する。
4.  **公開:** ユーザーがプレビューを確認して公開設定にし、一意のURL（パブリックリンク）を取得し、外部へ共有する。

## 4. 機能要件 (Functional Requirements)

### 4.1. ポートフォリオ設定・構築機能 (Frontend/Backend)

- ポートフォリオに含めるコンテンツ（セクション）を選択するUI。
  - **プロフィール設定:** 名前、アイコン、SNSリンク、簡単な自己紹介。
  - **自己PR / サマリ:** Step 3で生成（または自分で書いた）「これまでの道のり・アピールテキスト」。
  - **実績データ (Widget):** ダッシュボードから抽出された「全統合ヒートマップ」「言語別グラフ」「総コミット数」などのウィジェット化された要素を任意で追加。
  - **詳細なロードマップ/成果物一覧:** 特定のタスクの成果物リンク（リポジトリなど）へのアクセス。
- **レイアウトとデザイン:**
  - コンテンツをドラッグ＆ドロップで並び替えられるカスタマイズ機能。
  - LLMに対し「〇社の△△ポジション向けに響く構成でレイアウトを提案して」と指示し、セクションの配置や強調する要素をAIが組み替える機能。

### 4.2. プレビュー表示 (Frontend)

- 実際の公開ページを再現したWYSIWYG（What You See Is What You Get）プレビュー機能。
- モバイルビュー、デスクトップビューの切り替え表示。
- デザインテーマの切り替え（ダークモード/ライトモード、アクセントカラー、等）。

### 4.3. URL公開および制御機能 (Backend)

- ユーザーごとのユニークなパブリックURL（例: `logfo.app/user/portfolio_id_123` あるいはスラッグカスタム設定 `logfo.app/riku`）を発行する。
- **公開設定:** ドラフト（非公開）/ パブリック（公開）の切り替え。
- 特定の期間のみ有効なURLや、パスワード保護付きの共有リンク機能（将来的または有償版機能として検討）。
- 閲覧者（採用担当者等）がポートフォリオから簡単に「面談の申し込み」や「コンタクト」を取れるCTA（Call To Action）設定（オプション）。

## 5. 非機能要件 (Non-Functional Requirements)

- **パフォーマンス (SEO/OGP):** 公開されたポートフォリオは、採用担当者が共有しやすいよう適切なOGP画像・メタデータが生成される（SSR/SSGを用いたNext.jsでの静的出力・キャッシュ等によるLighthouseハイスコア推奨）こと。
- **レスポンシブデザイン:** 提出先企業がモバイルからアクセスしても綺麗に表示されるよう、強力なレスポンシブ対応を行う。

## 6. 特筆事項 / 検討課題

- ポートフォリオ機能の最初の段階ではデザイン（テーマ）は1種類に固定し、ユーザーがコンテンツを選ぶだけのシンプルな設定から始める。将来的には複数テンプレートやカスタムCSSの導入を検討する。
