# ポートフォリオ生成・共有機能 要件定義書 (Portfolio Generation Requirements)

## 1. 機能概要

本機能は、ユーザーがプロフィール情報・経歴・スキル・既存のサマリー/ロードマップを選択して、公開可能なポートフォリオページを生成し共有する機能です。

現行実装はMVPスコープを対象とし、外部ウィジェット統合やLLMによる自動レイアウト提案は将来拡張とします。

## 2. 対象ユーザー

- 就職・転職活動中のエンジニア（カジュアル面談等で企業に提出したいユーザー）
- これまでの軌跡を社内外（副業など）でアピールしたいユーザー
- Notionなどでポートフォリオを手作業で作ることに手間を感じているユーザー

## 3. ユースケース (User flow)

1. ユーザーがビルダー画面（`/portfolio`）を開く。
2. プロフィール、経歴ストーリー、スキル、表示するサマリー/ロードマップを選択・編集する。
3. 編集内容をリアルタイムプレビューで確認する。
4. 公開設定（Slug、公開/非公開）を設定して保存する。
5. 公開中の場合、`/portfolio/{slug}` を外部共有する。

## 4. 機能要件 (Functional Requirements)

### 4.1. ポートフォリオ設定・構築機能 (MVP)

- ポートフォリオに含めるコンテンツを設定できること。
- プロフィール設定:
  - 表示名（必須）
  - 自己紹介
  - アバターURL
  - SNSリンク（GitHub, X, Zenn, Qiita, AtCoder, Website）
- 経歴ストーリー設定:
  - 複数件の追加/編集/削除
  - 役割・職種、所属、期間、継続中フラグ、ストーリー本文
- スキル設定:
  - 文字列タグの追加/削除
- セクション設定:
  - 公開ページに表示するサマリーID配列
  - 公開ページに表示するロードマップID配列
- 保存時バリデーション:
  - Slug必須、英小文字/数字/ハイフンのみ、最大50文字
  - 表示名必須

### 4.2. プレビュー表示 (MVP)

- 実際の公開ビューと同一コンポーネントを利用したリアルタイムプレビューが行えること。
- 設定内容（プロフィール、経歴、スキル、選択セクション）が保存前でも即時反映されること。

### 4.3. URL公開および制御機能 (MVP)

- 公開URLは `logfo.app/portfolio/{slug}` 形式であること。
- Slugはユーザー間で一意であること（重複時は競合エラー）。
- 公開/非公開の切り替えが可能であること。
- 非公開ポートフォリオは公開取得APIでアクセスできないこと。

### 4.4. MVP対象外（将来拡張）

- 外部連携データのウィジェット表示（ヒートマップ、言語グラフ、総コミット数など）
- ドラッグ＆ドロップ並び替え
- LLMによるレイアウト提案・自動再構成
- モバイル/デスクトップのプレビュー切り替えUI
- テーマ切り替え（ライト/ダーク/アクセントカラーの明示的切り替え）
- 期限付きURL、パスワード保護リンク
- CTA（面談申し込み、コンタクト導線）設定

## 5. 非機能要件 (Non-Functional Requirements)

- レスポンシブデザイン:
  - モバイルおよびデスクトップで崩れずに閲覧できること。
- パフォーマンス/SEO:
  - MVPではクライアント取得を許容し、SSR/SSGおよび動的OGP生成は将来拡張とする。

## 6. 特筆事項 / 検討課題

- 現在は単一テーマでの提供を前提とする。
- 複数テンプレート、カスタムCSS、OGP画像自動生成は別フェーズで検討する。
